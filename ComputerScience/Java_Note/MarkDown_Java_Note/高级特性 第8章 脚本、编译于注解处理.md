## 8.3 使用注解

​	**注解是一种用于在源代码中嵌入元数据的语言机制，其语义由外部工具在编译期或运行时解析，用于实现代码生成、配置管理、行为增强等非侵入式编程目的。**

​	注解是一种让代码能够“自我介绍”的机制，它让框架和工具自动识别并处理代码，从而把程序员从繁琐的配置中解放出来。

比如：`@Override`能让编译器检查这个方法是不是真的重写了父类的方法

​	注解不会改变程序的编译方式。使用注解时，应选择对应的框架。



#### 8.3.1 注解简介 

​	注解是当做修饰符来使用的，它被置于被注解项之前。每一个注解的名称前面都加上了`@`符号。中间没有分号。例如Tomcat中的：

```java
@WebServlet()
```

​	下面是一个简单注解的示例：

```java
public class MyClass{
    ....
    @Test
    public void checkSomething()
}
```

​	注解`@Test`用于注解`checkSomething()`方法，`@Test`注解自身并不好做任何事情，它需要工具支持才会有用。例如，当测试一个类的时候，`JUnit`测试工具会调用所有标识为`@Test`的方法。

​	有些测试工具会认为测试代码，正式运行不需要，所以会直接把它从最终的程序包里删除掉。

​	

​	注解可以定义成包含元素的形式：

```java
@Test(timeout="10000")
```

​	这些元素可以被读取这些注解工具去处理。

​	除了方法外，还可以注解类、成员以及局部变量，这些注解可以存在与任何可以放置一个像`public`或者`static`这样的修饰符的地方。

​	每个注解都必须通过一个**注解接口**进行定。这些接口中的方法与注解中的元素相对应。例如，`Junit`的注解Test可以用下面这个接口进行定义：
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Test{
    long timeout() default 0L;
}
```

​	`@Interface`声明创建了一个真正的Java接口，处理注解的工具将接收那些实现了这个注解接口的对象。

​	注解`@Target`和`Retention`是元注解，它们注解了`Test`注解，`@Target(ElementType.METHOD)`将Test注解标识成一个只能运用到方法上的注解。`@Retention(RetentionPolicy.RUNTIME)`保证了当类文件载入虚拟机都时候，它任可以保留下来。





#### 8.3.2 示例：注解事件处理器

```java
myButton.addActionListener(()->doSomething());
```

​	上面的代码表示，为一个按钮，添加了一个事件，当你按下这个按钮后，会触发这个事件。这样很清晰，但是，当你设计了几十个，甚至几十个按钮时，结果可能会变成这样：

```java
btn1.addActionListener(() -> doBtn1());
btn2.addActionListener(() -> doBtn2());
btn3.addActionListener(() -> doBtn3());
// 再往下翻几十行……
btn50.addActionListener(() -> doBtn50());
```

​	UI代码变得臃肿，大量的`addActionListener`声明填充在构建UI的方法里。

​	在本节中，我们可以设计一个注解来免除这种苦差事。这个注解在下面的程序中使用方式如下：

​	当然，设计这种注解还是比直接使用`lambda`表达式要麻烦。但这种麻烦是对于框架开发人员来说的，对于使用框架的程序员来说，要方便上百倍。

```java
@ActionListenerFor(source="myButton")
void doSomething(){
    
}
```

​	对于使用注解的程序员来说，不在需要调用`addActionListener`了，哪个按钮想使用这个`doSomething`，直接在注解中的`source`指明按钮的id。

​	现在还需要定义一个注解接口

​	**这些注解本身不会做任何事情，它们只是存在于源文件中。**编译器将它们置于类文件中，并且虚拟机会将它们载入。现在需要的一个分析注解以及安装行为的监听器机制。这是我们设计的类`ActionListenerInstaller`的职责所在。`ButtomFrame`构造器将调用下面的方法：

```java
ActionListenerInstaller.processAnnotations(this); //this指向`ButtomFrame`
```

​	静态的`processAnnotations`方法可以枚举出某个对象接收到的所有方法，对于每个方法，它先获取`ActionListenerFor`注解对象，然后再对他进行处理：

```java	
Class<?> cl = obj.getClass(); //获取class对象
for(Method m: cl.getDeclaredMethods())
{
    ActionListenerFor a = m.getAnnotation(ActionListenerFor.class);
    if(a!=null)
}
```

​	这里使用了定义在`AnnotatedElement`接口中的`getAnnotation`方法，`Method、Constructor、Field、Class和Package`这些类都实现了这个接口。

​	源成员域的名字是存储在注解对象中的。我们可以通过`source`方法对它进行检索，然后找到匹配的成员域。

```java	
String fieldName = a.source();
Field f = cl.getDeclaredField(fieldName);
```

​	这表明我们的注解有一个限制条件，即源元素必须是一个成员域的名字，而不能是局部变量。

​	代码的剩余部分相当具有技术性。对于每一个被注解的方法，构造了一个实现`ActionListener`接口的代理对象。细节并不重要。**关键是要知道注解的功能是通过`processAnnotations`方法建立起来的**

​	

​	处理注解有三种不同的时机和层次。

- **运行时处理（本示例采用）**：通过**反射机制**，在程序跑起来（JVM 运行中）的时候，去读取类文件里的注解信息。这就是你刚才看到的 `ActionListenerInstaller` 里 `getAnnotation` 那一套代码。
- **源码级别处理**：在编译代码时，通过**源代码生成器**（Source-level annotation processing），直接在编译阶段生成额外的 Java 代码。比如自动生成 `addActionListener` 的代码放进源码里。
- **字节码级别处理**：在编译成 `.class` 文件后，用**字节码编辑器**直接修改 `.class` 文件中的指令，把 `addActionListener` 的调用硬塞进去。

​	

​	下图非常直观地展示了运行时处理注解的完整路径：

1. **源文件**：你在代码里写了 `@ActionListenerFor`。
2. **Java 编译器**：编译器把注解信息保留在 `.class` 文件里。
3. **类文件**：`.class` 文件中携带了 `@ActionListenerFor` 的数据。
4. **Java 虚拟机**：程序运行时，虚拟机加载 `ButtonFrame`。
5. **反射机制**：你的 `processAnnotations` 方法通过反射 API（`getAnnotation`等），读取存储在类文件里的注解信息，从而完成按钮事件的绑定。

下面给出具体示例程序；

*`ActionListenerInstaller`*

```java	
package runtimeAnnotations;

import java.awt.event.ActionListener;
import java.lang.reflect.*;
public class ActionListenerInstaller {

    public static void processAnnotations(Object obj){
        try{
            Class<?> cl = obj.getClass();
            for(Method m:cl.getDeclaredMethods()){
                ActionListenerFor a = m .getAnnotation(ActionListenerFor.class);
                if(a!=null){
                    //获取cl对象的指定的a.resource返回的字符串的字段
                    Field f = cl.getDeclaredField(a.source());
                    f.setAccessible(true);
                    addListener(f.get(obj),obj,m);

                }
            }
        }catch(ReflectiveOperationException e){
            e.printStackTrace();
        }
    }

    public static void addListener(Object source,final Object param, final Method m )
        throws ReflectiveOperationException
    {
        var handler = new InvocationHandler(){
            @Override
            public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                return m.invoke(param);
            }
        };

        Object listener = Proxy.newProxyInstance(null
        ,new Class[]{ java.awt.event.ActionListener.class},handler);
        Method add = source.getClass().getMethod("addActionListener", ActionListener.class);
        add.invoke(source,listener);
    }
}

```

## 8.4 注解语法

​	本节将介绍你必须了解的注解语法





#### 8.4.1 注解接口

​	注解是由注解接口来定义的：

```java	
modifiers @interface AnnotationName
{
    elementDeclaration1
	elementDeclaration2
	....
}
```

​	每个元素声明都具有下面这种形式：

```java	
type elementName();
```

​	或者

```java	
type elementName() default value;
```

​	例如，下面这个注解具有两个元素：`assignedTo`和`serverity`

```java	
public @interface BugReport{
    String assignedTo() default "[Noon]";
    int serverity();
}
```

​	所有的注解接口都隐式地扩展自`java.lang.annotation.Annotation`接口，`annotation.Annotation`是一个常规接口，而不是一个注解接口



​	你无法扩展注解接口（也就是被`@interface`声明的接口）。所有的注解接口都直接扩展自`java.lang.annotation.Annotation`，你也从来不用为注解接口提供实现类。

​	

​	注解接口的成员看起来很像方法，它们没有参数也没有`throws`子句，它们不是被`default`或`static`修饰，也不能有类型参数。不如说，把它称呼为**注解接口的元素**更为合适。

​	注解元素的类型为下列之一：

- 基本类型
- String
- Class(具有一个可选的类型参数，例如`Class<?> extends MyClass`、`String.class`)，不能是类对象
- enum类型
- 注解类型
- 由前面所诉类型组成的数组（不支持二维数组）

​	下面是一些合法的元素声明的例子：

```java	
public @interface BugReport{
    enum Status{UNCONFIRMED,CONFIRMED,FIXED,NOTABUG}; //枚举类型
    boolean showStopper() default false;//基本类型
    String assignedTo() default "[none]"; //String
    Class<?> testCase() default Void.class; //Class
    Status status() default Status.UNCONFIRMED; //枚举元素类型
    Reference ref() default @Reference(); //注解类型
    String[] reportedBy(); //数组类型
}
```



**`java.lang.annotation.Annotation`**

```java	
Class<? extends Annotation> annotationType()
    返回Class对象，它用于描述该注解对象的注解接口。调用注解对象上的getClass方法返回的才是真正的类，不是接口
boolean equals(Object other)
    如果other是一个实现了与该注解对象相同的注解接口的对象，并且如果该对象和other的所有元素彼此相等，那么返回true
int hashCode()
    返回一个与equals方法兼容，由注解接口名以及元素值衍生而来的散列码。
String toString()
    返回一个包含注解接口名以及元素值的字符串表示。
```





#### 8.4.2 注解

​	每个注解都具有下面这种格式：

```java	
@AnnotationName(elementName1 = value1,elementName2=value2,....);
```

​	例如：

```java	
@BugReport(assignedTo = "Harry", serverity =10)
```

​	元素的顺序无关紧要。如果某个的元素的值为指定，那么就使用声明的默认值，例如：

```java	
@BugReport(serverity=10)
```

​	元素`assignedTo`的值是字符串`[none]`。

​	

​	有两个特殊的快捷方式可以简化注解

​	如果没有指定元素，要么是因为注解中没有任何元素，要么是所有元素都使用默认值，那么你就不需要使用圆括号。例如：

```java	
@BugReport
```

​	和下面这个注解是一样的：

```java	
@BugReport(assignedTo="[none]",severity=0)
```

​	这样的注解又被称为标记注解。

​	另一种快捷方式是*单值注解*，如果一个元素具有特殊的名字`value`，并且在注解中没有指定其他元素。那么你就可以忽略掉这个元素名以及等号。例如，将前面的`ActionListenerFor`注解接口定义为如下形式：

```java	
public @interface ActionListenerFor{
    String value();
    ....
}
```

​	那么就可以将这个注解书写为如下形式：

```java	
@ActionListenerFor("yellowButton")
```

​	而不是：

```java	
@ActionListenerFor(value="yellowButton")
```

​	一个项可以有多个注解：

```java	
@Test
@BugReport(....)
public void checkRandomInsertions()
```

​	如果注解的作者将其声明为可重复的，那么你就可以多次重复使用一个注解：

```java	
@BugReport(...)
@BugReport(...)
public void checkRandomInsertions()
```

> ​	注释：因为注解是由编译器计算而来的，因此，所有元素值必须是常量

> ​	警告：一个注解元素永远不能设置为null，甚至不允许其默认值为null。你必须使用其他的默认值，例如“ ”或者Void.class。

​	

​	如果元素值是一个数组，那么要将它的值用括号括起来：

```java	
@BugReport(...,reportedBy = {"Harry","Carl"})
```

​	如果该元素具有单值，那么可以忽略这些括号：

```java	
@BugReport(...,reportedBy = "Harry")
```

​	



#### 8.4.3 注解各类声明

​	注解可以出现在许多地方。这些地方可分为两类：声明和类型用法。

​	声明注解可以出现在下列声明处：

- 包
- 类（包括enum）
- 接口（包括注解接口）
- 方法
- 构造器
- 实例域（包含enum常量）
- 局部变量
- 参数变量
- 类型参数

​	对于类和接口，主要将注解放置在`class`和`interface`关键词的前面：

```java	
@Entity public class User{
    ....
}
```

​	对于变量，需要把它放置在类型的前面：

```java	
@SupperssWarnings("unchecked") List<User> users =...;
public User getUser(@Param("id") String userId)
```

​	泛化类或方法的类型参数可以像下面这样被注解：

```java	
public class Cache<@Immutable V> {....}
```

​	





## 8.5 标准注解

​	`java.lang、java.lang.annotation`和`javax.annotation`包中定义了大量注解接口，其中四个是元注解，用于描述注解接口的行为属性，其他是规则接口。可以用它们来注解你的源代码中的项。

​		下表列出了这些注解

| 注解接口            | 应用场合                   | 目的                                                     |
| :------------------ | :------------------------- | :------------------------------------------------------- |
| Deprecated          | 全部                       | 将项标记为过时的                                         |
| SuppressWarnings    | 除了包和注解之外的所有情况 | 阻止某个给定类型的警告信息                               |
| SafeVarargs         | 方法和构造器               | 断言 varargs 参数可安全使用                              |
| Override            | 方法                       | 检查该方法是否覆盖了某一个超类方法                       |
| serial              | 方法                       | 检查该方法是不是正确序列化的方法                         |
| FunctionalInterface | 接口                       | 将接口标记为只有一个抽象方法的函数式接口                 |
| Generated           | 全部                       | 将项标记为由某个工具生成的源代码                         |
| Target              | 注解                       | 指明这个注解可以应用到哪些项上                           |
| Retention           | 注解                       | 指明这个注解可以保留多久                                 |
| Documented          | 注解                       | 指明这个注解应该包含在被注解项的文档中                   |
| Inherited           | 注解                       | 指明当这个注解应用于一个类的时候，能够自动被它的子类继承 |
| Repeatable          | 注解                       | 指明这个注解可以在同一个项上应用多次                     |



#### 8.5.1 用于编译的注解

​	`@Deprecated`注解可以被添加到任何不再鼓励使用的项上。所以，当你使用一个已过时的项时，编译器将会发出警告。这个注解与`Javadoc`标签`@deprecated`具有同样功效

​	`@SuppressWarnings`注解会告知编译器阻止特定类型的警告信息，例如，

​	`@SupperssWarnings("unchecked")`

​	`@Override`这种注解只能应用到方法上，编译器会检查具有这种注解的方法是否真正覆盖来着一个父类的方法。例如如果你声明：
```java
public MyClass{
    @Override
    public boolean equals(MyClass other);
}
```

​	那么编译器会报告一个错误。毕竟，这个`equals`方法没有覆盖`Object`类的`equals`方法。因为那个方法有一个类型为`Object`而不是`MyClass`的参数。

​	@`Generated`注解的目的是供代码生成工具来使用。任何生成的源代码都可以被注解，从而与程序员提供的代码区分开。



#### 8.5.2 元注解

​	`@Target`元注解可以应用于一个注解，以限制该注解可以应用到哪些项上。例如：

```java
@Target({ElementType.TYPE,ElementType.METHOD})
public @interface BurReport
```

​	下表列举了所有可能的取值情况，它们属于枚举类型`ElementType`。可以指定任意数量的元素类型，用括号括起来。

| 元素类型        | 注解适用场合                          | 元素类型       | 注解适用场合             |
| :-------------- | :------------------------------------ | :------------- | :----------------------- |
| ANNOTATION_TYPE | 注解类型声明                          | FIELD          | 成员域（包括 enum 常量） |
| PACKAGE         | 包                                    | PARAMETER      | 方法或构造器参数         |
| TYPE            | 类（包括 enum）及接口（包括注解类型） | LOCAL_VARIABLE | 局部变量                 |
| METHOD          | 方法                                  | TYPE_PARAMETER | 类型参数                 |
| CONSTRUCTOR     | 构造器                                | TYPE_USE       | 类型用法                 |

​	一条没有`@Target`限制的注解可以应用与任何项上。



​	`@Retention`元注解用于指定一条注解应该保留多长时间。其值见下表

| 保留规则 | 描述                                                         |
| :------- | :----------------------------------------------------------- |
| SOURCE   | 不包括在类文件中的注解                                       |
| CLASS    | 包括在类文件中的注解，但是虚拟机不需要将它们载入             |
| RUNTIME  | 包括在类文件中的注解，并由虚拟机载入。通过反射 API 可获得它们 |



​	`@Inherited`元注解只能应用于对类的注解。如果一个类具有继承注解，那么它的所有子类都自动具有同样的注解
